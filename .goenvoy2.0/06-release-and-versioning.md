# 06 · Release + versioning

## 6.1 Versioning model — keep per-module tags (ADR-0002)

goenvoy already uses the Go convention for multi-module monorepos: tags are of the form `<module-path>/v<major>.<minor>.<patch>`, e.g.:

- `arr/sonarr/v1.3.0`
- `metadata/video/tmdb/v0.4.2`
- `mediaserver/plex/v1.1.0`

This is **preserved** — it's the one correct way to do semver in a Go multi-module monorepo. Record it explicitly as ADR-0002.

The aggregate repo itself does **not** carry a `vX.Y.Z` tag. `CHANGELOG.md` at the repo root indexes releases by module + version, hand-curated (and optionally generated by release-please per-module in §6.3).

## 6.2 Per-module `release.yml` — UPGRADE existing

Current `release.yml` does: validate → GH Release with auto-generated notes. It's a fine skeleton but skips the supply-chain hardening golusoris ships. Upgrade to add:

- cosign keyless signature on `checksums.txt`.
- syft SPDX SBOM of the tagged source.
- `actions/attest-build-provenance` for SLSA-L3 provenance on release artefacts.
- Goreleaser (source-only mode — no binaries since goenvoy is a lib).

Replacement:

```yaml
name: Release

on:
  push:
    tags:
      - '**/v[0-9]+.[0-9]+.[0-9]+'
      - '**/v[0-9]+.[0-9]+.[0-9]+-*'

permissions:
  contents: read

jobs:
  release:
    name: Release tagged module
    runs-on: ubuntu-latest
    permissions:
      contents: write       # create GH Release + upload artefacts
      id-token: write       # cosign keyless + SLSA
      attestations: write   # attest-build-provenance
    steps:
      - uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6
        with:
          fetch-depth: 0

      - name: Parse tag
        id: tag
        run: |
          TAG="${GITHUB_REF#refs/tags/}"
          # tag shape: <module-path>/vX.Y.Z[-prerelease]
          MOD_PATH="${TAG%/v*}"
          VERSION="v${TAG##*/v}"
          echo "tag=$TAG" >> "$GITHUB_OUTPUT"
          echo "module=$MOD_PATH" >> "$GITHUB_OUTPUT"
          echo "version=$VERSION" >> "$GITHUB_OUTPUT"
          echo "dir=./$MOD_PATH" >> "$GITHUB_OUTPUT"

      - uses: actions/setup-go@4a3601121dd01d1626a1e23e37211e3254c1c06c # v6
        with:
          go-version: stable
          cache-dependency-path: ${{ steps.tag.outputs.dir }}/go.mod

      - name: Validate the tagged module
        working-directory: ${{ steps.tag.outputs.dir }}
        run: |
          go build ./...
          go vet ./...
          go test -race -count=1 ./...

      - name: Install cosign
        uses: sigstore/cosign-installer@398d4b0eeef1380460a10c8013a76f728fb906ac # v3

      - name: Install syft
        uses: anchore/sbom-action/download-syft@e22c389904149dbc22b58101806040fa8d37a610 # v0

      - name: Package source tarball
        id: pkg
        env:
          MOD: ${{ steps.tag.outputs.module }}
          VER: ${{ steps.tag.outputs.version }}
        run: |
          mkdir -p dist
          # Reproducible tarball (tar sorted, no mtimes, no owners).
          BASENAME="$(echo "$MOD" | tr '/' '_')_${VER}_source"
          (cd "${MOD}" && \
             find . -type f -not -path './coverage.out' -not -name '*.out' \
             | sort | LC_ALL=C tar --owner=0 --group=0 --numeric-owner \
             --mtime='@0' --sort=name --no-recursion -czf "../dist/${BASENAME}.tar.gz" -T - )
          (cd dist && sha256sum "${BASENAME}.tar.gz" > "${BASENAME}.checksums.txt")
          echo "basename=${BASENAME}" >> "$GITHUB_OUTPUT"

      - name: SBOM (syft → SPDX)
        run: |
          syft packages dir:${{ steps.tag.outputs.dir }} -o spdx-json > dist/${{ steps.pkg.outputs.basename }}.sbom.spdx.json

      - name: Sign checksums (cosign keyless)
        env:
          COSIGN_EXPERIMENTAL: "1"
        run: |
          cosign sign-blob --yes \
            --output-signature dist/${{ steps.pkg.outputs.basename }}.checksums.txt.sig \
            --output-certificate dist/${{ steps.pkg.outputs.basename }}.checksums.txt.pem \
            dist/${{ steps.pkg.outputs.basename }}.checksums.txt

      - name: Attest build provenance
        uses: actions/attest-build-provenance@a2bbfa25375fe432b6a289bc6b6cd05ecd0c4c32 # v4
        with:
          subject-path: |
            dist/${{ steps.pkg.outputs.basename }}.tar.gz
            dist/${{ steps.pkg.outputs.basename }}.checksums.txt
            dist/${{ steps.pkg.outputs.basename }}.sbom.spdx.json

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          tag_name: ${{ steps.tag.outputs.tag }}
          name: ${{ steps.tag.outputs.module }} ${{ steps.tag.outputs.version }}
          generate_release_notes: true
          files: |
            dist/${{ steps.pkg.outputs.basename }}.tar.gz
            dist/${{ steps.pkg.outputs.basename }}.checksums.txt
            dist/${{ steps.pkg.outputs.basename }}.checksums.txt.sig
            dist/${{ steps.pkg.outputs.basename }}.checksums.txt.pem
            dist/${{ steps.pkg.outputs.basename }}.sbom.spdx.json
```

Notes:

- No goreleaser. goenvoy ships no binaries, so goreleaser buys nothing. The reproducible tarball above is equivalent.
- cosign keyless uses GitHub OIDC — no long-lived key needed.
- Releases users can verify with `cosign verify-blob` (the command goes in `SECURITY.md` — see [03-governance-docs.md](03-governance-docs.md)).
- `actions/attest-build-provenance` emits the SLSA-L3 provenance automatically when `id-token: write` + `attestations: write` are set.

## 6.3 `release-please.yml` + `release-please-config.json` — OPTIONAL (phase 4)

release-please can run in **multi-package mode**, opening one release PR per module based on Conventional Commits scoped to that module. This replaces hand-curating `CHANGELOG.md`.

Decision: **adopt in phase 4**, not earlier. Reason: release-please multi-package mode requires a `release-please-config.json` enumerating every module (63 of them); writing that config is nice-to-have work, not blocking, and can be script-generated when the tooling is ready.

Skeleton:

`.github/workflows/release-please.yml`:

```yaml
name: release-please
on:
  push:
    branches: [main]
permissions:
  contents: read
jobs:
  release-please:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: googleapis/release-please-action@5c625bfb5d1ff62eadeeb3772007f7f66fdcf071 # v4.4.1
        with:
          config-file: release-please-config.json
          manifest-file: .release-please-manifest.json
```

`release-please-config.json` (generated — one package per module directory):

## 6.4 CHANGELOG — keep manual until phase 4

Current `CHANGELOG.md` is hand-maintained, Keep-a-Changelog format, per-module-header-grouped under a top-level release date. It's working and readable. Keep as-is through phases 1-3.

When release-please lands in phase 4, the root `CHANGELOG.md` will be **split** into one `CHANGELOG.md` per module directory, generated by release-please. The root `CHANGELOG.md` becomes an index.

## 6.5 `release-all.yml` — KEEP, extend minimally

Existing workflow validates + tags every module at a single version. Useful for coordinated bumps. Add the same cosign + syft + provenance steps as `release.yml` (or, simpler: have `release-all.yml` just push tags; then the `release.yml` workflow fires per tag and does all the heavy lifting).

**Recommended path**: keep `release-all.yml` as a thin *tag-creator* that fans out to `release.yml`. No duplication.

## 6.6 Sidecar: `tools/release-check.sh` (NEW)

Small pre-release audit, runnable locally:

```bash
#!/usr/bin/env bash
# Pre-release audit for one module.
#   usage: tools/release-check.sh arr/sonarr v1.4.0
set -euo pipefail
MOD="${1?module path}"
VER="${2?vX.Y.Z}"
cd "$MOD"
go test -race -count=1 ./...
go vet ./...
golangci-lint run --config "$(git rev-parse --show-toplevel)/.golangci.yml" ./...
govulncheck ./...
gosec -quiet ./...
# apidiff against previous
PREV=$(git tag --list "${MOD}/v*" --sort=-v:refname | head -1 || true)
if [ -n "$PREV" ]; then
  MI=$(go list -m)
  apidiff -m "$MI" . > /tmp/curr.txt 2>/dev/null || true
  SAVED=$(git rev-parse HEAD); git stash -q || true; git checkout -q "$PREV"
  apidiff -m "$MI" . > /tmp/prev.txt 2>/dev/null || true
  git checkout -q "$SAVED"; git stash pop -q || true
  apidiff /tmp/prev.txt /tmp/curr.txt && echo "OK compatible vs $PREV"
fi
echo "Ready to tag ${MOD}/${VER}"
```

Wired into the `/release-module` Claude skill in [07](07-claude-hooks-and-skills.md).
